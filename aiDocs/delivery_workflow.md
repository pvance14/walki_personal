# Walki Delivery Workflow

## Source-of-Truth Priority
1. `aiDocs/mvp.md` for near-term implementation scope.
2. `aiDocs/prd.md` for long-term product direction.
3. `aiDocs/architecture.md` for technical constraints and design boundaries.
4. `aiDocs/context.md` for agent-level operating guidance.

## Planning + Execution Workflow
1. Implement active phase based on roadmap checklist.
2. Keep roadmap status and tasks current during work.
3. Update `ai/changelog.md` with concise implementation summary and file references.
4. Move completed plan/roadmap pair to `ai/complete` after execution is complete.
5. Commit changes with a phase-specific commit message.

## Definition of Done per Phase
- All roadmap checklist items are complete.
- Completion signal in roadmap is satisfied.
- Changelog updated.
- Plan/roadmap pair moved to `ai/complete`.
