# Walki MVP Validation Runbook

## Objective
Run consistent stakeholder/user walkthroughs and capture feedback mapped to MVP validation goals.

## Session Format
1. Intro (2 min): explain demo scope and what is being validated.
2. Guided flow (8-10 min): landing -> quiz -> results -> demo -> waitlist.
3. Feedback capture (5 min): collect structured ratings and open comments.

## Structured Questions
- Persona appeal: Which persona felt most/least motivating?
- Quiz fit: Did your top personas feel accurate?
- Concept clarity: Could you explain Walki in one sentence?
- Emotional response: Did any message make you feel motivated or entertained?
- Shareability: Would you share your results or demo with someone?

## Data Capture Rules
- Use consistent note template per participant.
- Tag each finding by severity (`high`, `medium`, `low`) and validation goal.
- Keep evidence separate from interpretation.

## Iteration Loop
1. Aggregate findings after every 3-5 sessions.
2. Triage into: immediate fixes, post-MVP opportunities, out-of-scope.
3. Ship only high-impact in-scope fixes before next session batch.
